\section{绪论}

\subsection{研究背景}

近年来，大语言模型（Large Language Models, LLM）在问答、摘要、对话等任务中表现出强大的生成能力，但在知识截止、事实性与可控性方面仍存在不足，尤其是在需要严格依据外部证据的场景中容易产生“幻觉”\cite{gpt3, gpt4}。检索增强生成（Retrieval-Augmented Generation, RAG）通过将检索模块与生成模块结合，使模型在回答前能够从外部知识库中检索证据，从而提升事实性与可解释性\cite{ref1}。

然而，在真实使用中，RAG系统面临更复杂的输入分布：用户查询可能包含语义歧义（如“Python”既指编程语言也指蛇）、否定或约束条件（如“not named python”），也可能出现知识库中根本不存在答案的情况。若检索与重排序策略不能鲁棒处理上述场景，系统会返回不相关证据并诱发生成端的错误回答，从而影响用户体验与系统可信度\cite{ref26}。

\subsection{研究问题}

围绕“鲁棒、可解释、可部署”的目标，本文聚焦如下问题：
\begin{enumerate}
    \item 在有限规模语料上，如何构建一个端到端可复现的RAG实验平台，用于持续迭代与对比评估？
    \item 面对语义歧义与约束查询，如何通过重排序与结果融合提升Top-$k$结果的相关性？
    \item 面对无答案查询，如何设计安全机制降低高置信误检与无依据回答的风险？
\end{enumerate}

\subsection{本文工作}

针对上述挑战，本文实现并分析了一种基于置信度重检与后验验证的鲁棒问答系统。本文不再将RAG视为简单的单向流水线，而是尝试将其表述为一个兼顾正确率、部署成本与安全性的序列决策过程。具体而言，本文的工作包含以下四个方面：

\begin{itemize}
    \item \textbf{基于序列化对齐的多源数据入库}：系统采用序列化对齐策略，将文本段落、半结构化表格以及图谱三元组映射到统一的语义空间中进行混合检索，为复杂问答提供更广泛的基础线索。
    \item \textbf{基于置信度的二次检索触发策略}：以交叉编码器的重排序得分为启发式指标，系统能够在低置信度时主动触发二次扩展检索，而在高置信度时直接返回结果。该策略在多跳问题的证据覆盖率与系统平均延迟之间取得了有效的折中。
    \item \textbf{轻量级结构化证据整理}：系统将召回的碎片化文本按一定的实体和来源规则进行串联组织，形成逻辑更清晰的上下文，以支持后续大语言模型的回答生成。\textbf{需要强调的是，本文所指的“复杂推理”并非由系统内部的逻辑引擎进行显式的路径推导，而是指通过提升多跳证据的组织与供给质量，为大语言模型的隐式推理创造更好的上下文条件。}
    \item \textbf{引入事实核验的拒答机制}：本文在生成层之后接入后验验证模块，对大语言模型给出的回答声明进行证据比对。当检测到知识缺失或事实冲突时，系统强制返回“无答案”，从而有效阻断了幻觉向用户的传播。
\end{itemize}

本文在多跳基准数据集上对系统的各个机制进行了端到端的消融实验，并通过详细的误差类型拆解与显著性检验，客观分析了自适应检索与验证机制的真实贡献及其在复杂推理环境下的局限性。

\subsection{相关工作 (Related Work)}

近年来，检索增强生成（RAG）领域的发展经历了从“单一文本检索”到“主动式与验证式检索”的演进。本研究的方法论构建在以下四个维度的前沿探索之上：

\begin{itemize}
    \item \textbf{纯文本检索与重排序}：早期的RAG系统如DPR\cite{ref6}主要依赖稠密向量检索\cite{bm25}。为解决语义歧义，后续研究引入了基于BERT的交叉编码器（Cross-Encoder）进行精确的段落重排序\cite{bert_rerank}。然而，这些系统通常仅支持非结构化文本，在面对需要从表格或知识图谱中提取信息的结构化查询时往往束手无策。
    \item \textbf{异构知识与多模态 RAG (Heterogeneous \& Multi-modal RAG)}：面对真实场景中的多模态数据，部分研究尝试在 RAG 中融合结构化数据。例如，HybridQA\cite{hybridqa} 和 OTT-QA\cite{ottqa} 探索了在文本与表格上的联合推理；而 Think-on-Graph\cite{ref17} 和 HippoRAG\cite{ref14} 则尝试将知识图谱引入检索过程。尽管如此，如何在一个统一的特征空间中低成本地联合召回这三类异构数据，仍是一个尚未完全解决的工程难题。
    \item \textbf{主动式与多跳检索 (Active \& Multi-hop Retrieval)}：在处理HotpotQA\cite{ref22}等复杂推理任务时，一次性检索往往难以召回完整的证据链。近期的前沿研究开始探索让大模型主动控制检索时机。例如，FLARE\cite{ref12}通过预测未来生成的低置信度词元来触发检索；IRCoT\cite{ref26}则交替执行思维链推理与检索步骤。然而，此类方法往往忽略了频繁调用大模型带来的高昂Token成本与推理延迟。
    \item \textbf{自我反思与防幻觉机制 (Self-Reflection \& Anti-Hallucination)}：为缓解生成端幻觉，Self-RAG\cite{ref8}通过特殊的反思标记（Reflection Tokens）训练模型对检索结果进行自我批评；CoVe (Chain-of-Verification)\cite{ref24}则提出先生成初始草稿，再提取独立事实声明进行逐步验证。这些验证方法虽然有效，但大多依赖模型自身的隐式知识，缺乏对外部检索“证据链（Evidence Chain）”显式追溯机制，容易在知识库存在盲区时产生“虚假确信”\cite{ref25}。
\end{itemize}

与上述工作相比，本文更强调把RAG系统写成“可分析、可审视、可解释”的决策流程，而不仅仅是若干工程模块的叠加。具体而言，本文关注的不是单纯报告某个模块带来的数值提升，而是解释这些提升分别来自检索补全、来源感知重排序与后验验证约束中的哪一部分，以及它们在成本、安全与覆盖率之间形成了怎样的可部署权衡。
